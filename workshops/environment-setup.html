Getting Started: Software

The Maroon's Data & Technology team uses a variety of software to produce high-quality, interactive data visualizations. This document will introduce you to our development environment and guide you through downloading the necessary software. These tools can be quite complex, so don't worry if you don't get it right away. No prior experience is required, and please feel free to reach out to us at data@chicagomaroon.com if you have any questions!

The Terminal

The Terminal is a program on your computer designed to let you interact with files, code, programs, and more at a low level. You do all of this by running commands. Thus, the Terminal is an example of a Command Line Interface (CLI).

The Terminal works within your computer's file system. When you open it, it will display a mostly blank screen with a single line where you can type. This line will look different depending on what type of computer you're using, but it will always display a location in your file system. This is your "working directory" ("directory" is a synonym of "file" in this context).

Try typing pwd and hitting "enter". This stands for "Print Working Directory", and it should have printed a line of text showing your current location in more detail.

Now, try ls! This command, short for "List", prints the contents of your current working directory. For example, if I ran ls in my "Maroon_Data_Viz" folder, the Terminal would list folders like "2024", "node_modules", and "template_visual", along with files like "CONTRIB.md", "example-graphs.html", and "package.json". If your working directory is empty, it will print nothing.

The final command you need in order to navigate the Terminal is cd, or "Change Directory". As its name suggests, This command lets you change your working directory. If my working directory is "2024" in the Maroon file system, I can navigate to a sub-folder using cd [folder-name] (for example, cd fgli-map). I can also navigate to my working directory's parent folder using cd ...

Using pwd, ls, and cd, you can go anywhere in your computer's file system! Take a few minutes to mess around with these commands to make sure you're comfortable with them (don't worry, you won't break anything). The Terminal can do much more, but these are the three basic commands you'll need for now. Later, we'll go through some commands specific to software we use.

Code Editors

As the name suggests, code editors are apps that let you edit code on your computer. Like Word, they allow you to open and edit text-based documents.

For a popular, free, all-purpose code editor, try Visual Studio Code, or VSCode (download here). If you want an editor that's more specific to web development (which is much of what we do here), try Webstorm (download here), a powerful JavaScript IDE (integrated development environment). This will offer all sorts of configuration options as well as thorough documentation for any and all relevant features. You can get a free student license as a student at UChicago.

Git, GitHub, and GitHub Desktop

Git is the most widely used version management software. Once you download Git, you can set up a "Git repository" in one of your existing folders. This will allow you to run Terminal commands to track your changes, create and switch between different versions of your code (called "branches"), and cleanly combine changes made on different branches. For more detailed information about Git, see this book.

Please download Git here: https://git-scm.com/downloads

GitHub is a widely used system that extends Git to allow for collaboration on a Git repository. Our team hosts its code in a collection of public repositories found here. Through GitHub, you can download (or "clone") a new repository, make changes, and upload (or "push") those changes. When you clone a repository, you tell Git to copy the repository's contents into a specified folder on your computer. Later, you can "pull" others' changes to the shared ("remote") repository, and when you edit the code (and save your changes) in your "local" repository, you can push them to the "remote". You can also create and publish branches for others to see/edit.

Please create a GitHub account at github.com. You can use your personal and/or UChicago email address, but be aware that if you only connect your UChicago account, your GitHub may disappear after graduation. When you are done, please send us your username so we can add you as a contributor.

Although you can use Git and GitHub through commands in the Terminal, we don't recommend using them that way. Instead, you can download an app called "GitHub Desktop" to perform all of the Git/GitHub tasks.

Please download GitHub Desktop here. Then, clone our Data Visualization repository by going to "Current Repository > Add > Clone Repository > URL", entering the URL "https://github.com/chicagomaroon/data-visualizations.git", and choosing an empty folder on your computer where you want the new repository to go.

When you want to contribute to our repository, create a branch using the "Current Branch" menu in GitHub Desktop, make your changes using your favorite code editor, select the files whose changes you want to push using the left-hand-side menu, commit those changes (using present imperative tense to title your Commit), and push those changes using the top-right button. Then, go to our repository in GitHub and open a pull request asking to merge your branch into the main one. This will start a conversation in GitHub where admins can provide feedback, approve your changes, and merge your branch. We will review this process again in a future workshop.

Node.js

Node.js is a complex piece of software that allows us to use custom scripts to perform specific tasks in our repository. For example, if you run npm run format ("NPM" = "Node Package Manager") in the Terminal in our repository, Node will run a program on the entire repository that applies a standard format to everything. These scripts are an integral part of our team's workflow. We'll go through them in more detail later.

To download Node.js, go here. NOTE: Please download Node version 20.9.0 by running nvm install v20.9.0 instead of nvm install 22.

HighCharts

HighCharts is a JavaScript library used to make high-quality, interactive data visualizations that can be embedded into websites with relatively little effort. See this page for examples of what HighCharts can do. Most of our graphs are made through HighCharts. We will discuss how to make graphs like this in future workshops.

You don't need to download anything to use HighCharts.

Review

Now, you should understand the basics of navigating the Terminal, how to use a code editor, how Git and GitHub work, and what we use Node and HighCharts for. You also should have successfully downloaded VSCode or Webstorm, Git, GitHub Desktop, and Node. To check VSCode/Webstorm and GitHub Desktop, find those apps on your computer. To check Git, go to the Terminal (it doesn't matter what your working directory is) and run git --version. To check Node and NPM, run node --version and npm --version, respectively. If any of these commands give you an error, please reach out to us for assistance.
